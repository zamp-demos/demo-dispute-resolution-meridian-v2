⚠ Representment Rejected

Visa rejected the merchant's representment on March 3, 2026. The chargeback stands in favor of the cardholder.

Rejection Reason

Insufficient Compelling Evidence for CE 3.0 Rule 10.4.3

The merchant submitted frictionless 3D Secure (3DS) authentication as compelling evidence, but Visa determined this was insufficient for the following reasons:

  • IP address geolocation mismatch was not addressed in the representment
  • Transaction originated from IP address in Kiev, Ukraine while cardholder's billing address is Seattle, WA
  • No evidence provided to explain geographic discrepancy
  • Frictionless 3DS alone does not satisfy CE 3.0 requirements when IP geolocation contradicts cardholder location

What Was Submitted

Merchant's representment included:

  • ✓ Frictionless 3DS authentication log (transaction approved without challenge)
  • ✓ Device fingerprint matching prior transaction from same account
  • ✗ No IP geolocation analysis or explanation
  • ✗ No additional fraud prevention data

Visa Ruling Summary

Per CE 3.0 Rule 10.4.3, frictionless 3DS is compelling evidence ONLY when all authentication factors align with the cardholder's profile. The IP geolocation mismatch (6,200 miles from billing address) raises legitimate fraud concerns that were not rebutted.

Decision: Chargeback upheld. Merchant liable for $1,180.00.
